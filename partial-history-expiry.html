1. Introduction

Ethereum has stored every transaction since 2015, which has caused its blockchain data to grow extremely large—over 1 terabyte (TB). This makes it harder for people to run their own nodes, as the storage requirements are high. To address this, Ethereum developers introduced Partial History Expiry (PHE), a feature that reduces node size while keeping the network secure.

2. What Is Partial History Expiry?

Partial History Expiry allows Ethereum clients (like Geth, Besu, Nethermind, Erigon, and Reth) to delete block bodies and receipts from before the Merge (September 2022) while still keeping all block headers and the current state:

  • Removed: Old transactions and receipts from before the Merge.
  • Kept: All block headers, balances, and all data after the Merge.

This frees up 300–500 GB of disk space, making it much easier to run a full Ethereum node on a normal hard drive. Importantly, the removed history is not lost—it can still be downloaded from public archives (ERA1 format) if needed.

3. Why Partial History Expiry Matters

Partial History Expiry has several important benefits for Ethereum:

  • Lighter Nodes: Running a node now requires less storage space, making it more accessible.
  • More Decentralization: Because storage is lighter, more people can run nodes, not just institutions with powerful hardware.
  • Better Efficiency: Syncing and maintaining nodes becomes faster and smoother.
  • Security Maintained: All cryptographic block headers and current state are preserved, so the integrity of Ethereum is not compromised.
  • Foundation for the Future: This is the first step toward rolling history expiry, where old data will automatically expire to prevent endless growth.

4. What It Means for Ethereum

Partial History Expiry is a milestone for Ethereum’s sustainability. By making nodes lighter, it:

  • Keeps Ethereum accessible to a wider community of users.
  • Strengthens decentralization, since more people can participate in running nodes.
  • Improves long-term scalability, ensuring the blockchain does not grow too large to handle.
  • Protects security, since essential data is preserved and old history is still retrievable through archives.

In simple words, this upgrade makes Ethereum lighter, cheaper, and more decentralized, while preparing the network for the future.

5. Conclusion

Partial History Expiry solves a critical storage problem in Ethereum by pruning old, pre-Merge data while keeping the blockchain secure and verifiable. It reduces node sizes by hundreds of gigabytes, increases accessibility, and strengthens decentralization. For Ethereum, this marks an important step in becoming more efficient, more sustainable, and more inclusive, without sacrificing its security or history.
